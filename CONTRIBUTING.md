# Contributing

This tutorial is **community-driven** — open to anyone who wants to add an example, fix content, or propose a new topic.

## Two ways to start

**Visual:** open [`/contribute`](https://learn-ai-native-dev.example/contribute) on the live site for an interactive picker.

**Text:** read [`.github/CONTRIBUTING.md`](.github/CONTRIBUTING.md) for the full guide, including the three contribution shapes (Tiny / Medium / Larger), agent slash-prompts, the CLA, and licensing.

## At a glance

| Want to… | How |
|---|---|
| Add a project example | [Add an example](.github/ISSUE_TEMPLATE/add-example.yml) issue → run `/add-example` |
| Fix a typo or improve a sentence | Just open a PR (or [Improve content](.github/ISSUE_TEMPLATE/improve-content.yml)) |
| Report stale info | [Report stale content](.github/ISSUE_TEMPLATE/report-stale-content.yml) issue |
| Propose a new module or path | [Propose a topic](.github/ISSUE_TEMPLATE/propose-topic.yml) issue → run `/propose-topic` |

See [`.github/CONTENT_LIFECYCLE.md`](.github/CONTENT_LIFECYCLE.md) for how community contributions get promoted to `official`.
